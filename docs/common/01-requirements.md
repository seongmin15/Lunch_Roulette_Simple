# 요구사항

---

## 1. 핵심 목표

| 목표 | 측정 기준 | 우선순위 |
|------|----------|---------|
| 현재 위치 기반 주변 식당 자동 수집 | 앱 실행 후 30초 이내에 반경 내 식당 목록이 표시된다. | P0 |
| 가격·거리 필터 적용 후 룰렛 실행 | 필터 설정부터 룰렛 결과 표시까지 5초 이내 완료된다. | P0 |
| 최근 룰렛 결과 10건 히스토리 저장 | 앱 재실행 후에도 최근 10건의 결과가 유지된다. | P1 |

---

## 2. 성공 시나리오

점심시간에 앱을 열고 필터를 설정한 뒤 룰렛을 돌려 30초 이내에 오늘의 점심 식당이 결정된다.


---

## 3. 비목표

| 비목표 | 근거 | 재검토 시점 |
|--------|------|-----------|
| 앱 스토어 출시 및 다중 사용자 지원 | 개인 사용 목적이므로 배포 및 다중 사용자 관리가 불필요하다. |  |
| 소셜 기능(공유, 친구 추천 등) | 1인 사용자 앱으로 소셜 기능은 범위를 벗어난다. |  |
| 식당 리뷰 작성 및 평점 기능 | 외부 지도 API의 리뷰 데이터를 활용하는 것으로 충분하다. |  |

---

## 4. 범위

### 4.1 포함 (In-Scope)

| 기능 | 사용자 스토리 | 우선순위 | 복잡도 |
|------|-------------|---------|--------|
| 현재 위치 기반 주변 식당 목록 조회 | 사용자로서 앱을 열면 내 위치 주변의 식당 목록을 자동으로 볼 수 있다. | must | M |
| 가격대·거리 필터 설정 | 사용자로서 예산과 이동 거리를 설정하여 원하는 조건의 식당만 룰렛에 포함할 수 있다. | must | S |
| 식당 룰렛 실행 및 결과 표시 | 사용자로서 룰렛 버튼을 누르면 필터링된 식당 중 하나가 무작위로 선택된다. | must | M |
| 최근 룰렛 결과 10건 히스토리 | 사용자로서 최근에 룰렛으로 선택된 식당 10곳을 확인할 수 있다. | should | S |
| 식당 상세 정보 표시 | 사용자로서 선택된 식당의 이름, 주소, 거리, 영업시간을 확인할 수 있다. | should | S |

### 4.2 제외 (Out-of-Scope)

| 기능 | 제외 사유 | 예정 시점 |
|------|----------|----------|
| 앱 스토어 배포 | 개인 사용 목적이므로 배포가 불필요하다. | backlog |
| 소셜 공유 기능 | 1인 사용자 앱 범위를 초과한다. | v2 |
| 식당 직접 예약 연동 | 초기 버전에서는 정보 조회 및 선택에 집중한다. | v2 |

---

## 5. 가정

| 가정 | 틀리면? | 검증 계획 |
|------|--------|----------|
| 카카오 지도 API 또는 네이버 지도 API를 통해 주변 식당 데이터를 무료 또는 저비용으로 조회할 수 있다. | Google Places API 등 대체 API를 사용하거나, 유료 플랜으로 전환해야 한다. |  |
| 사용자는 스마트폰의 위치 권한을 앱에 허용한다. | 수동으로 위치를 입력하는 폴백 기능을 추가해야 한다. |  |
| 개인 사용 목적이므로 API 일일 호출 한도를 초과하지 않는다. | API 호출 결과를 로컬에 캐싱하여 호출 횟수를 줄여야 한다. |  |
